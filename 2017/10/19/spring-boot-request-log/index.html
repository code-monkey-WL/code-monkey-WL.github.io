spring boot 打印请求日志

Posted on 2017-10-19 In spring

在业务开发中，日志在项目有着非常重要的作用，业务统计、性能分析、错误排查都是需要日志的场景，正确的日志 log 是一个优秀的项目必不可少的一部分。在几年的开发工作中，我见过有些项目没有关键的业务日志和请求日志，在排查异常的时候就比较麻烦，难以复现。在这里简单的分享一下如何在 web 开发中简便的记录 http request 和 response 日志。项目中使用的开发框架是 spring boot 。

为什么要打印请求日志

请求日志记录了一个请求的请求方式、请求头、请求参数以及服务端返回的响应信息，从请求发起到 服务端响应结束，整个生命周期都记录了下来，这样在排查问题等情景下，客户端给了什么参数，服务端返回了什么值，就可以一目了然。

如何打印

在项目中打印请求日志有很多种方法，最初级的是在每个 controller 的各个方法中打印日志，我见过有人这样做，代码中到处都是log.info('request param {}',params) ，这种打印日志会把业务逻辑和请求日志混合在一起，不便于后期维护，而且不够整洁，当修改了一个方法的参数或返回时，还需要修改对应的 log 信息。

spring 提供了一个 ContentCachingRequestWrapper，可以使用它来缓存 request body 的数据，它的作用

caches all content read from the {@linkplain #getInputStream() input stream} and {@linkplain #getReader() reader} and allows this content to be retrieved via a {@link #getContentAsByteArray() byte array}.

需要注意的是，当 stream 未被读取的时候，它是不会将 stream 的内容写到 content 中的。所以在 request 被处理前，调用 #getContentAsByteArray() 得到仍然是一个空数组。
