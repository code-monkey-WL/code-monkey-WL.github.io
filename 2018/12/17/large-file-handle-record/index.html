Posted on 2018-12-17 In java

最近负责部门一个比较老的项目，这个项目为各个业务线提供服务，它使用了一个 redis 集群来缓存一些用户数据，可能是因为机器服役太久了？其中一台 redis 服务器一天晚上挂掉了，而由于历史遗留问题，这个 redis 既没有做主从，也没有过备份，这就导致了整个内存中的数据都挂掉了。这对业务线产生了一部分影响，为了降低影响，业务线的同事从之前的历史日志里回溯了 8G 的历史数据，希望我把这些数据重新导入到 redis。

总结

  • 合理的切分文件，多线程并发处理，可以加快处理大文件的速度。
  • 大量的数据先考虑根据特征过滤掉无用的数据，减少实际处理的数据量。
  • 运行稳定的老项目架构也有风险点，有些坑不能因为运行稳定就忽略掉，应该及时处理。
  • 重要的数据一定要做持久化，redis 集群一定要做主备，且开启 AOF/RDB 备份
