X-Forwarded-For 伪造 IP 漏洞

Posted on 2018-05-08 In java

最近接手的一个项目在某天早上收到了一条来自公司安全组的漏洞警告：用户可以伪造请求头来伪造 IP，绕过 IP 相关的限制策略。

漏洞原因

为了项目安全，针对同一个 IP 的频繁访问有一定的策略限制，那么如何获取 IP 呢？

在 Java 项目获取客户端 IP ，可以通过 HttpServletRequest#getRemoteAddr 方法获取，这个方法可以直接获取到连接服务器的 client 的 IP，因为 HTTP 在传输层是 TCP 协议，它获取的实际是 TCP 层连接的 client 端的 IP。但是在项目有代理服务器的时候，这个获取到的是代理服务器的 IP 地址。有一段常用的获取 IP 的代码如下，我的这个项目里也使用了：

X-Forwarded-For

这个请求头并不在 Http 的标准定义中，它最早是由 Squid 这个代理服务器的开发人员引入的，后来成为了事实上的标准，各大 HTTP 代理、负载均衡等转发服务广泛使用，后来被写入 RFC 7239（Forwarded HTTP Extension）标准之中。

X-Real-IP

一般我们的 nginx 都会如下配置：

通过配置请求头中的 X-Real-IP ，那么就可以通过request.getHeader("X-Real-IP") 获取 client 的 IP 地址，这个同样只要在最外层的 nginx 代理上配置。这里 X-Real-IP 是个自定义的请求头，并不在标准中，也可以改成别的名字。

这里我获取了 X-Real-IP 来作为 IP 地址，推荐与安全有关的场景，只使用 X-Real-IP；在与安全无关的场景，例如通过 IP 提供城市服务、天气信息、旅游指南等，可以从 X-Forwarded-For 靠前的位置获取 IP，但是需要校验 IP 格式合法性。
